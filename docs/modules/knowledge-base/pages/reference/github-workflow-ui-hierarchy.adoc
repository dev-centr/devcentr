= GitHub Actions workflow chooser: hierarchy inconsistency

User intent: *"Find a template"* (any source).  
Current UI: two paths with different visibility and misleading wording.

**Note:** The UI is hard to understand. GitHub mixes "workflow templates" (full files) and "Marketplace actions" (steps you call with inputs) without making the difference clear. There is no competent visual design system; information is thrown at you with no clear order. For the full picture and what we use instead, see xref:github-actions-templates-vs-marketplace.adoc[GitHub Actions: Templates vs Marketplace].

== Current state (problem)

  B --> B1["Templates shown in same dialog"]
  C --> C1["Link leads to Marketplace"]

  subgraph prominence["Prominence / valence"]
    B1 --- high["High — visible by default"]
    C1 --- low["Low — hidden; label suggests manual setup, not more templates"]
  end

  style B fill:#e8f5e9
  style C fill:#ffebee
  style C1 fill:#ffcdd2
  style low fill:#ffcdd2
----

*Why it’s inconsistent:*
* Same user goal (“pick a template”), but two different valence levels.
* “Set up a workflow yourself” sounds like *no* template — so users don’t discover the Marketplace.
* Community/Marketplace templates are de facto “second class” despite being the same kind of content (templates).

== Desired state (same valence)

[source,mermaid]
----
flowchart LR
  subgraph desired["Single dialog, equal valence"]
    A["Choose a workflow"]
    A --> B["Starter workflows"]
    A --> C["Marketplace / community workflows"]
  end

  B & C --- same["Same dialog, same prominence — user sees all templates in one place"]
  style B fill:#e8f5e9
  style C fill:#e8f5e9
----

*Fix:* Expose Marketplace in the *same* dialog as starter workflows so both are “choose a template” options at the same level. No separate path that implies “I’ll do it myself.”
