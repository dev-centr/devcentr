== Automated Releases with GitHub Actions

Git hosting platforms can automatically build your releases and attach binaries. This saves you from manually compiling and uploading files.

=== Is it Expensive?

**Usually no.**

* **Public repositories**: GitHub Actions is typically **free**.
* **Private repositories**: You get a monthly quota of minutes (enough for many release builds).

=== How to Automate Release Builds

You need a workflow file in `.github/workflows/`.

This workflow will:

1. Trigger when you push a tag starting with `v` (e.g., `v0.1.0`).
2. Spin up a runner (Linux, Windows, macOS).
3. Install your compiler/toolchain.
4. Build your project.
5. Upload the binary to the Release page automatically.

==== Example Workflow (`.github/workflows/release.yml`)

[source,yaml]
----
name: Release

on:
  push:
    tags:
      - 'v*' # Trigger on tags like v1.0.0

permissions:
  contents: write # Required to create releases

jobs:
  build:
    name: Build ${{ matrix.os }}
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]

    steps:
      - uses: actions/checkout@v4

      # Install your toolchain here (example: for a compiled language)
      # - name: Install toolchain
      #   run: |
      #     echo "Install compiler or runtime"

      - name: Build
        run: |
          echo "Run your build command here"
          # e.g. npm run build, cargo build --release, dub build --build=release, etc.

      - name: Upload binaries to release
        uses: softprops/action-gh-release@v1
        if: startsWith(github.ref, 'refs/tags/')
        with:
          files: |
            path/to/your/binary*
          name: ${{ github.ref_name }} # e.g. v0.1.0
          body: "Automated release build."
          draft: false
          prerelease: false
----

Adjust the **toolchain install** and **build** steps to match your language and project.

=== Key Takeaway

By adding a release workflow, you turn your \"Source Ready\" project into a \"Binary Ready\" project automatically every time you push a tag.
